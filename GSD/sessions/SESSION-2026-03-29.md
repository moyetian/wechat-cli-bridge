## Key Learnings

1. `M003` 的命令面可以先于真实邮箱 UAT 落地。
2. 邮件附件无需重新发明本地文件准备流程，复用 `stageLocalMedia` 就够。
3. 当前真正的 blocker 只剩一套真实 SMTP 凭据和收件箱验证。
4. `STATE.md` 的切片状态不能先于 milestone 文档实际落盘，否则会造成记录失真。
5. `templates/config.example.json` 很容易和 `setup.ts` 生成的默认配置漂移，需要在 release gate 专门复核。
6. 对部分收件端，`/mailhtml` 只发 HTML 不如 multipart 稳，带 plain-text fallback 更稳妥。
7. `/mailfile` 的真实问题往往不是 staging 本身，而是用户输入路径与 bridge 解析路径不一致，因此回显 resolved path 很重要。
