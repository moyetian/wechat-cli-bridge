## Key Learnings

1. **权限模式目前是伪模式**
   - `/permission` 只改 session 状态
   - agent 执行层未消费 permission mode
   - CLI 参数仍默认带 bypass 标志

2. **测试失败不是业务逻辑问题，而是初始化副作用问题**
   ```typescript
   fs.ensureDirSync(LOGS_DIR);
   export const storage = new Storage();
   ```

3. **下一阶段必须先做底座**
   - 没有可移植测试，就没有可靠 CI
   - 没有真实权限协议，就不能声称 bridge 已具备权限控制

4. **GSD 规划需要从“功能列表”升级为“里程碑 + slice + 验证门”**
   - 每个 slice 必须有 must-haves
   - 每个 slice 必须有明确验证命令
   - 每个 slice 必须有依赖顺序
