## Key Learnings

1. **S03 的主要问题不是“不能发”，而是“发出去以后不够可用”**
   - 命令面缺显式图片入口
   - 失败反馈粒度太粗
   - 默认安全边界不存在

2. **quoted path 支持是命令面可用性的关键**
   - 没有它，带空格文件名在真实使用里几乎不可控

3. **把媒体发送结果从 `boolean` 提升为结构化结果后，bridge 才能给出可操作反馈**
   - `NOT_FOUND`
   - `PROTECTED_PATH`
   - `UNSUPPORTED_IMAGE_TYPE`
  - `TOO_LARGE`
  - `UPLOAD_FAILED`
  - `SEND_FAILED`

4. **`aes_key` 的编码细节是这次设备端可用性的关键**
   - 仅仅“上传成功 + sendmessage 成功”并不代表手机端可打开
   - 需要与官方插件保持完全一致的 `aes_key` 编码格式
